## 4. Baseline Detection Queries

This section formalises baseline monitoring queries into structured detection use cases. These detections are intended to validate monitoring capability rather than simulate full attack scenarios.

---

### 4.1 Failed Logon Threshold Detection (Event ID 4625)

Objective: Identify repeated failed authentication attempts from a single source.

```spl
index=win11_endpoint_logs EventCode=4625
| stats count by Account_Name, Source_Network_Address
| where count >= 5
| sort -count
```

This detection highlights accounts experiencing multiple failed logon attempts, which may indicate brute force behaviour or password spraying.

### 4.2 Privileged Logon Monitoring (Event ID 4672)

Objective: Monitor elevated sessions granted special privileges.

```spl
index=win11_endpoint_logs EventCode=4672
| table _time Account_Name Logon_Type
| sort -_time
```

Event ID 4672 indicates that administrative privileges were assigned to a logon session. Monitoring this event supports visibility of elevated access activity.

### 4.3 Process Execution Monitoring (Sysmon Event ID 1)

Objective: Identify execution of potentially sensitive processes.

```spl
index=win11_endpoint_logs "powershell.exe"
| table _time host Image CommandLine
| sort -_time
```

This detection validates process execution telemetry captured via Sysmon. Monitoring command-line activity supports visibility into potentially suspicious process behaviour.
