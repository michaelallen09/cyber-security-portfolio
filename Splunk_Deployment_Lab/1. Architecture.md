# 1. Architecture

## 1.1 Overview

This lab establishes a working SIEM deployment using Splunk Enterprise and a Windows endpoint within a virtualised environment.

The objective of this phase is to configure log ingestion and confirm baseline visibility.

## 1.3 Lab Components

The environment consists of:

- Windows 11 endpoint  
- Splunk Universal Forwarder  
- Splunk Enterprise (Ubuntu 22.04)  
- Host machine for web access  

---

## 1.4 Log Flow

1. Windows generates Security and Sysmon events.  
2. The Universal Forwarder collects the events.  
3. Logs are sent to Splunk over TCP 9997.  
4. Events are indexed into `win11_endpoint_logs`.  
5. Searches are performed via Splunk Web (TCP 8000).

---

## 1.5 Component Summary

### Windows 11 Endpoint
- Generates Security and Sysmon logs

### Universal Forwarder
- Monitors WinEventLog
- Forwards logs to TCP 9997

### Splunk Enterprise
- Receives logs on TCP 9997
- Indexes into `win11_endpoint_logs`
- Provides web interface on TCP 8000
