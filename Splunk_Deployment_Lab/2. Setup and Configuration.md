# 2. Setup and Configuration

## 2.1 Splunk Enterprise Installation

Splunk Enterprise was installed on an Ubuntu 22.04 virtual machine.

Configuration steps included:

- Installing the Splunk Enterprise package
- Starting the Splunk service
- Enabling the web interface (TCP 8000)
- Enabling the receiving port (TCP 9997)

A dedicated index was created:

- Index name: `win11_endpoint_logs`
- Data type: Events

This index stores all Windows endpoint telemetry for this lab.

---

## 2.2 Universal Forwarder Configuration

The Splunk Universal Forwarder was installed on the Windows 11 endpoint.

Configuration included:

- Setting the forward server to the Splunk indexer (TCP 9997)
- Configuring monitoring for:
  - Windows Security Event Log
  - Microsoft-Windows-Sysmon/Operational

The forwarder was verified to be connected and actively forwarding events.

---

## 2.3 Sysmon Deployment

Sysmon was installed on the Windows endpoint to enhance telemetry collection.

Sysmon was configured to capture:

- Process creation events
- Network connections
- System activity relevant to detection use cases

This provides improved visibility beyond standard Windows Security logs.

---

## 2.4 Service Validation

The following checks were performed:

- Confirmed Splunk service was running on Ubuntu
- Confirmed Splunk Web was accessible on TCP 8000
- Confirmed receiving port TCP 9997 was active
- Confirmed forwarder was running on Windows
- Verified events were being indexed into `win11_endpoint_logs`

These steps ensured the ingestion pipeline was functioning before moving to validation and detection testing.
