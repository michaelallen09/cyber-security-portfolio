## 2. Setup and Configuration

### 2.1 Splunk Enterprise Installation

Splunk Enterprise was installed on an Ubuntu 22.04 virtual machine within VirtualBox.

Configuration steps included:

- Installing the Splunk Enterprise package  
- Starting the Splunk service  
- Enabling the web interface (TCP 8000)  
- Enabling the receiving port (TCP 9997)  

Service status was verified:

```bash
sudo /opt/splunk/bin/splunk status
sudo ss -lntp | grep 9997
```

### 2.2 Universal Forwarder Configuration

The Splunk Universal Forwarder was installed on the Windows 11 endpoint.

<img width="621" height="483" alt="Screenshot 2026-02-22 214006" src="https://github.com/user-attachments/assets/968d256b-d19a-4a01-893c-43a357b6b22d" />

The forward server was configured to point to the Splunk indexer (TCP 9997).

Forwarder connectivity was verified on the Windows 11 endpoint:

```powershell
.\splunk.exe list forward-server
```

The output confirmed active forwarding to the Splunk server.

### 2.3 Sysmon Deployment

Sysmon was installed on the Windows endpoint to improve telemetry visibility.

```powershell
get-service sysmon64
```
